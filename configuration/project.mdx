---
title: Project configuration
description: Reference for all retype.yml project-level configuration options, including branding, navigation, search, theme, and integrations.
---

Retype reads the **retype.yml** file for instructions on how to configure and build your project. The file is typically placed in the root of your project.

<Note>
When running `retype start`, any changes to **retype.yml** are automatically detected and the website is regenerated in the browser. If you started the server with `retype serve`, you need to run `retype build` manually and refresh the browser.
</Note>

The **retype.yml** file is optional but recommended. If you run `retype start` without one, Retype will auto-generate a basic **retype.yml** you can then customize. You can also run `retype init` to generate it explicitly.

## Sample retype.yml

```yml retype.yml
input: .
output: .retype

url: docs.example.com

start:
  pro: true  # Try Retype Pro features locally

branding:
  title: Project Name
  label: Docs

links:
  - text: Getting Started
    link: https://retype.com/guides/getting-started/

footer:
  copyright: "&copy; Copyright {{ year }}. All rights reserved."
```

## input

<ParamField path="input" type="string" default=".">
  Custom path to the input directory. The path is relative to the **retype.yml** location.

  ```yml
  input: ./src
  ```
</ParamField>

---

## output

<ParamField path="output" type="string" default=".retype">
  Custom path to the output directory. The path is relative to the **retype.yml** location.

  ```yml
  output: ./docs
  ```

  <Note>The `output` directory is only used by `retype build`. When using `retype start`, files are stored in memory and not written to disk.</Note>
</ParamField>

---

## url

<ParamField path="url" type="string">
  The base URL of your website. Retype uses this to construct absolute links and generate a CNAME file.

  ```yml
  url: docs.example.com
  ```

  Subdomains, paths, and explicit protocols are all supported:

  ```yml
  url: example.com/docs
  url: http://example.com/docs/
  url: companyx.github.io/docs  # GitHub Pages without custom CNAME
  ```

  If no protocol is supplied, Retype assumes `https`.
</ParamField>

## branding

Branding configuration for your Retype generated website.

<ParamField path="branding.title" type="string">
  The main text title shown in the upper-left corner of the generated website. Can be used alongside `logo` and `logoDark`.

  ```yml
  branding:
    title: Example.com
  ```
</ParamField>

<ParamField path="branding.label" type="string" default="Docs">
  Optional logo label text rendered to the right of the title or logo in the upper-left corner.

  ```yml
  branding:
    label: Docs
  ```
</ParamField>

<ParamField path="branding.logo" type="string">
  Path to a logo image file relative to [`input`](#input), or an inline `<svg>` string. Default is `null`.

  ```yml
  branding:
    logo: static/logo.png
  ```
</ParamField>

<ParamField path="branding.logoDark" type="string">
  Path to a dark mode logo image relative to [`input`](#input), or an inline `<svg>` string. Default is `null`.

  ```yml
  branding:
    logo: static/logo.png
    logoDark: static/logo-dark.png
  ```
</ParamField>

<ParamField path="branding.logoWidth" type="number">
  Custom width (in pixels) for the branding logo. If not set, Retype auto-detects the intrinsic width.

  ```yml
  branding:
    logo: ./logo.svg
    logoWidth: 150
  ```
</ParamField>

<ParamField path="branding.logoHeight" type="number">
  Custom height (in pixels) for the branding logo. If not set, Retype auto-detects the intrinsic height.

  ```yml
  branding:
    logo: ./logo.svg
    logoHeight: 31
  ```
</ParamField>

<ParamField path="branding.logoAlign" type="string" default="left">
  Logo alignment relative to the `title`. Accepted values: `left`, `right`.

  ```yml
  branding:
    logo: static/logo.png
    logoAlign: right
  ```
</ParamField>

<ParamField path="branding.baseColor" type="string" default="#5495f1">
  **Retype Pro only.** Sets the main brand color as a shorthand for `theme.base.base-color`.

  ```yml
  branding:
    baseColor: "#8839ef"
  ```
</ParamField>

<ParamField path="branding.colors.label" type="object">
  Override text and background colors for the branding label badge.

  | Option | Type | Description |
  | --- | --- | --- |
  | `text` | `string` | Label text color |
  | `background` | `string` | Label background color |

  ```yml
  branding:
    colors:
      label:
        text: "#ffffff"
        background: "#ff0000"
  ```
</ParamField>

## links

Custom links added to the top-bar navigation of all pages.

```yml
links:
  - text: Getting Started
    link: https://retype.com/getting_started/
```

<ParamField path="links[].text" type="string">
  The link text label.
</ParamField>

<ParamField path="links[].link" type="string">
  The URL for the link. Accepts a `.md` filename, internal path, or external URL.
</ParamField>

<ParamField path="links[].icon" type="string">
  An icon for the link. Accepts an Octicon name, emoji shortcode, `<svg>` element, or path to an image file.

  ```yml
  links:
    - text: Issues
      link: https://github.com/retypeapp/retype/issues/
      icon: bug
  ```
</ParamField>

<ParamField path="links[].iconAlign" type="string" default="left">
  Position of the icon relative to the link text. Accepted values: `left`, `right`.
</ParamField>

<ParamField path="links[].target" type="string">
  Sets the `target` attribute of the hyperlink. Use `blank` to open in a new tab (Retype automatically transforms `blank` → `_blank`).
</ParamField>

<ParamField path="links[].title" type="string">
  Sets the `title` attribute on the hyperlink, shown as a browser tooltip on hover.
</ParamField>

<ParamField path="links[].items" type="list">
  Nested menu items that render the parent link as a dropdown in the header navigation. Dropdown menus are supported one level deep. Each nested item supports the same properties as top-level links.

  ```yml
  links:
    - text: Guides
      items:
        - text: Installation
          link: /guides/installation.md
          icon: download
        - text: Configuration
          link: /configuration/project.md
  ```
</ParamField>

        link: https://github.com/retypeapp/retype/discussions/

## meta

<ParamField path="meta.title" type="string">
  A site-wide suffix appended to the HTML `<title>` element of all pages.

  ```yml
  meta:
    title: " | Example.com - Widgets for the internet"
  ```

  With an `About us` page, the resulting title would be:
  ```html
  <title>About us | Example.com - Widgets for the internet</title>
  ```
</ParamField>

<ParamField path="meta.siteName" type="string">
  Sets the value for the `<meta property="og:site_name">` tag.

  ```yml
  meta:
    siteName: Retype
  ```
</ParamField>

## edit

Enables and customizes **Edit this page** links on content pages.

<ParamField path="edit.repo" type="string">
  The repository URL where source files are hosted. Setting this value enables **Edit this page** links.

  ```yml
  edit:
    repo: "https://github.com/<your-organization>/<your-repo>/"
  ```

  To link directly to the edit view:

  ```yml
  edit:
    repo: "https://github.com/<your-organization>/<your-repo>/edit/"
  ```
</ParamField>

<ParamField path="edit.base" type="string">
  Optional base path to a subdirectory within the repository where the `.md` source files are stored.

  ```yml
  edit:
    repo: "https://github.com/your-org/your-repo"
    base: /src/docs
  ```
</ParamField>

<ParamField path="edit.branch" type="string" default="main">
  The branch to point to within the repo.

  ```yml
  edit:
    repo: "https://github.com/your-org/your-repo"
    branch: master
  ```
</ParamField>

<ParamField path="edit.label" type="string" default="Edit this page">
  Custom label text for the edit link.

  ```yml
  edit:
    repo: "https://github.com/your-org/your-repo"
    label: Edit on GitHub
  ```
</ParamField>

## nav

Navigation configuration for the left sidebar.

<ParamField path="nav.mode" type="string" default="default">
  **Retype Pro only.** Controls how the sidebar navigation is structured.

  | Option | Description |
  | --- | --- |
  | `default` | Expandable/collapsible tree structure |
  | `stack` | Top-level folders are always expanded, creating a stacked layout |

  ```yml
  nav:
    mode: stack
  ```
</ParamField>

<ParamField path="nav.icons.mode" type="string" default="all">
  **Retype Pro only.** Controls how icons are displayed in the left sidebar.

  | Mode | Description |
  | --- | --- |
  | `all` | Show icons for all navigation items |
  | `folders` | Show icons only for folder items |
  | `pages` | Show icons only for page items |
  | `top` | Show icons only for top-level items |
  | `none` | Hide all navigation icons |

  ```yml
  nav:
    icons:
      mode: none
  ```
</ParamField>

## toc

**Retype Pro only.** Controls the right-sidebar Table of Contents.

<ParamField path="toc.depth" type="string | number" default="2-3">
  The heading depth range to include in the Table of Contents.

  | Value | Description |
  | --- | --- |
  | `2` | H2 headings only |
  | `2-3` | H2 to H3 (default) |
  | `1-4` | H1 to H4 |
  | `2,4` | H2 and H4 only |

  ```yml
  toc:
    depth: 2-3
  ```
</ParamField>

<ParamField path="toc.label" type="string">
  Custom label for the top of the Table of Contents column.

  ```yml
  toc:
    label: On this page
  ```
</ParamField>

## search

Customization options for the website search component.

<ParamField path="search.mode" type="string" default="full">
  The search index creation mode.

  | Mode | Description |
  | --- | --- |
  | `full` | Full-text index of all headings and content |
  | `partial` | All headings plus first paragraph under each heading |
  | `basic` | All headings plus only the first paragraph per page |

  ```yml
  search:
    mode: partial
  ```

  <Tip>If search feels slow on a large project, try `partial` or `basic` mode.</Tip>
</ParamField>

<ParamField path="search.hotkeys" type="list" default="k">
  Keyboard key(s) to focus the search field.

  ```yml
  search:
    hotkeys:
      - "/"
  ```
</ParamField>

<ParamField path="search.maxResults" type="number" default="20">
  Maximum number of search results to display.
</ParamField>

<ParamField path="search.minChars" type="number" default="2">
  Minimum number of characters required before a search query is run.
</ParamField>

<ParamField path="search.placeholder" type="string" default="Search">
  Placeholder text displayed in the search input.
</ParamField>

<ParamField path="search.noResultsFoundMsg" type="string" default="Sorry, no results found.">
  Message shown when no results are returned.
</ParamField>

<ParamField path="search.excludeCode" type="boolean" default="false">
  Exclude code blocks from the search index.

  ```yml
  search:
    excludeCode: true
  ```
</ParamField>

<ParamField path="search.exclude" type="list">
  **Retype Pro only.** Exclude pages or folders from the search index while still generating them in the website. Uses the same wildcard patterns as [`exclude`](#exclude).

  ```yml
  search:
    exclude:
      - /blog
      - /draft
  ```
</ParamField>

<ParamField path="search.preload" type="boolean" default="false">
  Preload the search index on page load. Useful when combined with `generator.directoryIndex.append: true` for offline filesystem browsing.
</ParamField>

## theme

**Retype Pro only.** Customize your site's visual appearance by overriding CSS design variables.

See the full [theme variables reference](/configuration/theme-variables) for all available variables.

<ParamField path="theme.base" type="object">
  Light mode theme variables. All theme variables can be customized here.

  ```yml
  theme:
    base:
      base-color: "#8839ef"
      base-bg: "#eff1f5"
      primary: "#1e66f5"
      success: "#40a02b"
      danger: "#d20f39"
      warning: "#df8e1d"
  ```
</ParamField>

<ParamField path="theme.dark" type="object">
  Dark mode theme variables. These override `theme.base` variables when dark mode is active.

  ```yml
  theme:
    dark:
      base-color: "#ca9ee6"
      base-bg: "#303446"
  ```
</ParamField>

## include and exclude

<ParamField path="exclude" type="list">
  Exclude files or folders from the build output. Uses `.gitignore`-style patterns with support for `?`, `*`, `**`, and `!`.

  ```yml
  exclude:
    - "draft/"
    - "*_temp/"
    - "/src/temp.md"
  ```

  <Note>Files and folders prefixed with `.` or `_`, and any `node_modules` folder, are excluded by default.</Note>
</ParamField>

<ParamField path="include" type="list">
  Explicitly include files or folders that would otherwise be excluded. Retype automatically includes common image formats, PDFs, ZIPs, and the `static/`, `public/`, `assets/`, and `resources/` folders.

  ```yml
  include:
    - "*.py"
    - "demo*.js"
    - "**/www/**"
  ```
</ParamField>

## locale

<ParamField path="locale" type="string" default="en">
  The primary language for the generated website. Retype supports 24 languages. Provide an ISO language code.

  ```yml
  locale: fr
  ```

  Supported codes include: `ar`, `da`, `de`, `el`, `en`, `es`, `fi`, `fr`, `he`, `hi`, `hu`, `hy`, `it`, `ja`, `ko`, `nb`, `nl`, `pt`, `pt-PT`, `ro`, `ru`, `sv`, `tr`, `zh`, and more.
</ParamField>

<ParamField path="locale.dateFormat" type="string">
  Configure how dates are rendered across the site, including blog dates and the Last Updated component.

  ```yml
  locale:
    dateFormat: MM-dd-yyyy
  ```

  Supported specifiers: `yyyy`, `yy`, `MMMM`, `MMM`, `MM`, `M`, `dddd`, `ddd`, `dd`, `d`.
</ParamField>

## scheme

**Retype Pro only.** Control the default color mode for the website.

<ParamField path="scheme.mode" type="string" default="system">
  Sets the default color scheme.

  | Mode | Description |
  | --- | --- |
  | `system` | Follow the visitor's system preference (default) |
  | `dark` | Always render in dark mode |
  | `light` | Always render in light mode |

  ```yml
  scheme:
    mode: dark
  ```

  <Note>If a visitor manually selects a color mode using the toggle, their choice overrides this setting.</Note>
</ParamField>

## serve

Configuration for the built-in Retype development web server.

<ParamField path="serve.host" type="string" default="localhost">
  Host address for the local development server. A custom port can be appended with a colon.

  ```yml
  serve:
    host: 127.0.0.1
    # or with a port:
    # host: 127.0.0.1:5005
  ```
</ParamField>

<ParamField path="serve.port" type="number" default="5001">
  Port for the local development server. If the port is in use, Retype auto-increments unless a value is explicitly set.

  ```yml
  serve:
    port: 5005
  ```
</ParamField>

## templating

<ParamField path="templating.enabled" type="boolean" default="true">
  Enable or disable the Retype content templating engine project-wide.
</ParamField>

<ParamField path="templating.liquid" type="boolean" default="false">
  Enable Liquid `{% ... %}` syntax. Note: incompatible with GitBook-style component syntax.
</ParamField>

<ParamField path="templating.loopLimit" type="number" default="1000">
  Maximum loop iterations allowed in templates. Set to `0` to disable the limit (not recommended).
</ParamField>

<ParamField path="templating.namespace" type="string">
  Optional namespace prefix for template variables. When set, variables are accessed as `{{ namespace.project.branding.title }}`.
</ParamField>

## poweredByRetype

<ParamField path="poweredByRetype" type="boolean">
  **Retype Pro only.** Set to `false` to remove the "Powered by Retype" branding from the footer.

  ```yml
  poweredByRetype: false
  ```
</ParamField>
